# 智能数据瞭望系统 — 任务清单与分工

> 双人并行开发任务清单。按「用户前端体验」(A 线) vs「管理后台数据」(B 线) 一刀切分，A 不动后端 schema，B 不碰对话流，唯一共享点是数字员工表（事先约定 A 只读 B 独占写入）。

---

## 0. 项目现状速览

- **技术栈**：FastAPI + SQLAlchemy + SQLite + Tailwind 单 HTML 多页（无前端工程化）。
- **已有基础**：21 个 HTML 页面、20 个 controller、20+ 个 model、OpenAI 协议客户端（支持 function calling）、数据采集模型、敏感词过滤器、Milvus 可选。
- **关键约束**：
  - `ai_model.model_type` 当前仅 `chat/embedding/rerank`，需扩展到 `image/video`。
  - 各 HTML 页面 `sidebar` 当前硬编码，已有 `static/js/app.js` 统一侧边栏注释但未落地。
  - 种子数据散落在 `main.py` 的 `migrate_*` 函数里，未收敛。

## 6. Git 协作约定

- `main`：只接可运行版本，底座完成后合入。
- `feat/A`（用户前端）：A 选手独占。
- `feat/B`（管理后台）：B 选手独占。
- **-shared 文件**：`models/agent.py`、`models/skill.py`、`models/message.py`（如有改动）需事先约定，A 不可改 schema 只读，B 独占写入。
- **合并节奏**：底座每天合一次 `main` → 各 feature 分支 rebase。
